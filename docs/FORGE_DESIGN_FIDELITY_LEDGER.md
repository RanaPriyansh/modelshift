# FORGE Design Fidelity Ledger

## Reference and decision

The behavioral reference is [forge-design-lab.priyansh-rana.chatgpt.site](https://forge-design-lab.priyansh-rana.chatgpt.site/). The selected interaction direction remains **The Vanishing Instrument**:

> Claim → plausible models → point of disagreement → separating test → unaided proof

The reference is a design fixture. Synthetic evidence, fixture permissions, and example numerical claims from it are not production truth.

## Five-part comparison

| Dimension | Reference contract | Candidate implementation | Status |
| --- | --- | --- | --- |
| Hierarchy | One dominant question per viewport; physical or source object before explanation | Four Worlds begin with a mystery/claim; Studio begins with one learning question and assembles the instrument beside it | **Preserved in Worlds; partial in the denser Studio form** |
| Composition | Cinematic opening → brief two-reading compiler → dense instrument only when needed → quiet proof | Each working World follows that sequence; Studio preview shows the reusable compiler anatomy without pretending to run proof | **Preserved structurally** |
| Typography | System sans, large tightly tracked hero, small monospaced semantic labels | Studio uses the reference system-sans/mono hierarchy up to 104 px; older home/account and legacy physics surfaces retain Georgia in places | **Partial; known visual debt** |
| Color | paper `#f7f4ed/#ebe6dc`, ink `#080d12/#0d141b/#141e27`, cyan `#64d5d2`, amber `#f2b84b`, violet `#a88cff` | Studio adopts the exact core palette; Worlds retain semantically compatible earlier cyan/amber/violet variants | **High semantic fidelity, incomplete token unification** |
| Interaction and motion | Deterministic physical time; movement only for causality/state/withdrawal; reduced-motion alternatives | Physics state, vectors, graphs, and numbers share deterministic samples; stage changes, proof withdrawal, keyboard focus, and reduced-motion paths are tested | **Preserved** |

## Production fidelity decisions

- Amber remains learner-authored or committed material.
- Violet remains uncertain interpretation.
- Cyan remains experiment/evidence transformation.
- Exactly two readings are enforced in the Lesson Studio schema and the primary-source compiler.
- AI is a bounded interpretation/draft layer, not a persistent avatar, chat companion, grader, or proof authority.
- Proof removes hints and instructional AI but does not remove keyboard, motion, or semantic access controls.
- No gradients, glass, particles, badges, streaks, points, or celebratory correctness were added.
- Provider setup lives in Studio rather than a persistent chat rail; the active learning World remains primary.

## Browser evidence

- Studio inspected at 1440×900 and 320×800; no horizontal overflow.
- Full Playwright matrix: 42 passed, 16 intentional cross-project skips, 0 failed.
- Studio-specific desktop/mobile matrix: 12 passed, including key non-retention, four-provider selection, strict draft rendering, two readings, separating test, bounded transfer, and a forced 320 px check.
- Primary Source World independently passed full Chromium journey, 320 px, focus, image load, reduced motion, and exhaustive validator checks.
- Production nonce CSP was inspected in the rendered HTML: framework scripts and the response policy carried the same request nonce, and React controls hydrated.

## Known deviations

1. The universal home and account surfaces still use portions of the earlier dark/serif visual system instead of the design lab's paper-led, system-sans shell.
2. The design lab's full frontier map, return-proof scheduling surface, guide relationship UI, and project World are not implemented as production features.
3. The Studio is an authoring/draft instrument; its right-hand compiler preview is intentionally denser than a learner World and is not itself evidence of learning.
4. Cloud identity, permissions, counters, and evidence timelines in the design reference are not copied as fixtures; real persistent state is required before those surfaces can ship.
5. A formal external accessibility audit and representative learner comprehension test remain open.

The release may claim design-direction and interaction fidelity for the implemented slices. It may not claim the entire reference gallery is operational or that visual similarity validates educational outcomes.
