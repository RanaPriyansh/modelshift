## Brand hierarchy

- **FORGE** is the learner-facing learning operating system.
- **ModelShift** is the invariant learning protocol: commit a model, meet evidence, reconstruct, and prove without help.
- A **Learning World** is a domain-specific implementation of that protocol.

The permanent trust line is:

> Learner acts. AI assists. Evidence decides.

FORGE must feel like a public learning institution rebuilt with care: rigorous, calm, humane, inspectable, and usable by a curious child or an adult. It must not feel like an LMS, chatbot, game economy, productivity dashboard, or AI companion.

## The product-defining sequence

Every complete Learning World follows the smallest compatible version of:

> Claim -> plausible models -> point of disagreement -> separating test -> reconstruction -> unaided proof -> bounded evidence

The World progressively assembles only the instruments required to challenge the learner's model, then removes assistance. The emotional arc is curiosity -> commitment -> legibility -> agency -> reconstruction -> productive exposure -> earned evidence.

### Scene ownership

1. **Cinematic Mystery** owns Encounter and Commit. Begin with an event, object, source, performance, or problem—not a lesson title or dashboard.
2. **Mental Model Canvas** owns one brief compiler moment. Preserve learner-authored language, show at most two plausible readings, identify the disagreement, and select the smallest test.
3. **Scientific Instrument** owns Test and Explore. The medium that can disagree—simulation, exact arithmetic, source set, executable code, rubric, observation, or human review—is primary.
4. **Quiet Proof** owns withdrawal and evidence. Assistance visibly leaves. Access features remain. The result states only what this attempt demonstrated and what remains untested.

Only the compiler may branch spatially. Only the experiment may expose dense instrumentation. Only proof may remove instructional support.

## Shell versus World

The FORGE shell supports discovery, broad foundations, learner questions, projects, people, evidence, return proofs, and guide controls. It never becomes the learning experience itself.

The first viewport asks one dominant question:

> What do you want to understand, make, or become able to do?

Catalogs, foundations, and programmes appear below that invitation. Inside a World, global navigation recedes and the World returns to one consequential action per viewport.

The shell may show evidence states such as `Building`, `Proved once`, `Ready to revisit`, `Carried into a project`, and `Open question`. It never displays mastery percentages, XP, streaks, leaderboards, comparative rank, or engagement rewards.

## Semantic color

Color carries stable authorship and authority:

| Color | Meaning |
| --- | --- |
| Amber/gold | Learner-authored question, prediction, explanation, or committed work |
| Violet | AI interpretation, suggestion, translation, or other disclosed model contribution |
| Cyan/teal | Tested consequence, reviewed evidence, executable result, or source transformation |
| Warm orange | Human feedback, review, or observation |
| Quiet paper | Protected proof state in which instructional AI is structurally absent |

Do not use color as the only carrier of meaning. Every graph, vector, source state, and proof condition needs a label, number, table, or text equivalent.

## Typography and surfaces

- Editorial serif headings pose the central question and mark moments of reflection.
- Modern sans serif text carries actions and explanations.
- Monospace labels identify evidence, provenance, versions, modes, and measured state.
- Dark midnight navy is the working surface; warm ivory is the default foreground.
- Surfaces are flat, bordered, and spatially precise. No glass, gradients, ambient particles, or decorative glow.
- Density may increase only when the learner has asked for instrumentation or provenance.

## AI transparency

AI is a bounded interpretation layer, never a character.

Each World exposes `How this world knows` and an AI action ledger:

```text
AI used in this step
- Rephrased your question
- Matched your explanation to an authored possibility
- Suggested one reviewed source

AI did not
- Decide the result
- Write your proof
- Change the standard
```

At protected proof:

```text
Help is now off

You will get one new case.
No hints, replay, or AI will appear until you submit.
```

Accessibility, translation, input method, and assistive technology remain available in proof. The product withdraws instructional advantage, not access.

## Motion

- Physical or executable state uses deterministic time.
- Interface motion communicates causality, state change, or withdrawal.
- Compiler connectors draw once.
- Synchronized representations update on the same frame.
- Scene transitions move no more than 24 CSS pixels.
- Reduced motion removes decorative transitions while preserving paused, stepped, numerical, and textual evidence.
- No autoplay or forced pacing.

## Responsive rules

Desktop keeps the active World canvas near 1180 CSS pixels. The shell uses generous single-question composition before any catalogue grid.

Mobile:

- Replace a row of numbered dots with `Step N of 7 - Stage` and one progress bar.
- Hide bottom/global navigation inside an active World.
- Recompose scientific scenes; never scale and crop desktop SVG labels.
- Put legends and textual alternatives below the world.
- Use dependency lists instead of shrinking a map canvas.
- Use single-column evidence timelines and labeled accordions.
- Preserve 44 CSS pixel targets, 16 CSS pixel inputs, keyboard support, reduced motion, and non-drag alternatives.

## Copy rules

- Use short, concrete questions and explicit verbs.
- Describe interpretations as uncertain possibilities, never diagnoses.
- Keep learner language verbatim and visibly learner-owned.
- Explain why a test separates models before naming a rule.
- Describe support neutrally and record it without punishment.
- Scope evidence to this task, representation, assistance condition, and time.
- Name what remains open, including delayed retention.
- Never say `mastered` from one response.

## Anti-patterns

- Persistent chat, AI avatar, emotional simulation, or anthropomorphic thinking narration.
- Dashboard-first opening, course chrome, content feed, badges, streaks, points, or mastery scores.
- Probabilities attached to provisional learner interpretations.
- Freeform canvas or dragging as the only path to understanding.
- Hidden controls, forced pacing, autoplay, or narration-dependent meaning.
- Celebratory correctness, shame, punitive help language, or generic `Continue` actions.
- Generated activities presented as reviewed, verified, or source-grounded.
- Withdrawing access features during proof.

## Review gates

A complete World is not ready until:

1. The compiler mechanism is understandable without narration.
2. Every representation derives from one authoritative state or reviewed source package.
3. The learner commits before any answer-bearing support.
4. AI actions and non-actions are inspectable.
5. Proof mode has no hint, replay, or instructional-AI path.
6. The result distinguishes demonstrated evidence from untested claims.
7. The full flow works by keyboard and at 320 CSS pixels.
8. Reduced motion preserves operation and evidence.
9. A learner can reject or correct a provisional interpretation.
10. The World remains fully usable on its authored fallback path.
